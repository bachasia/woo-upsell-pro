# Phase 02 — Embedding Generator + Storage

**Status:** pending
**Priority:** High (blocker for phase 03, 04)
**Depends on:** Phase 01

## Overview

Generate embeddings for products and store as `_wup_embedding` post_meta.
Trigger on product save (incremental) and via admin batch (initial setup).
Text input = title + tags + attributes + category + short description (all configurable).

## Related Code Files

- **New:** `includes/ai/class-wup-product-embedder.php` — core embed + store logic
- `includes/class-wup-plugin.php` — register save hook

## Storage Schema

```
post_meta key:   _wup_embedding        — JSON-encoded float[] (1536 elements, ~6 KB)
post_meta key:   _wup_embedding_model  — string, e.g. "text-embedding-3-small" (for versioning)
```

The `_wup_embedding_model` key is critical: when the admin switches to a different model,
similarity search must skip vectors generated by a different model (incompatible dims/space).

## Text Input Strategy

Build embed text from (in priority order — labels help model attention):

```
"{title}. Tags: {tags_csv}. Attributes: {attrs_csv}. Category: {leaf_category}. {short_desc_stripped}"
```

| Signal | Setting toggle | Notes |
|---|---|---|
| Title | always on | Most specific signal |
| Tags | `wup_ai_embed_use_tags` (default: on) | Curated, high signal |
| Attributes | `wup_ai_embed_use_attributes` (default: on) | Critical for variable products |
| Category | `wup_ai_embed_use_category` (default: on) | Use leaf-most category only |
| Short description | `wup_ai_embed_use_description` (default: on) | Strip HTML, append last |

- Strip all HTML tags before concatenating
- Trim to max ~2000 chars (~512 tokens) to stay within model limits
- If a field is empty/disabled, omit that segment entirely (no empty labels)

**Variable products:** Embed the parent product only. Merge all variation attributes
(e.g., Size: S/M/L/XL, Color: Red/Blue) into the attributes segment.

## Implementation Steps

1. **Create `WUP_Product_Embedder`** (`class-wup-product-embedder.php`):
   - `static build_embed_text( int $product_id ): string`
     - Compose text from title + tags + attributes + category + short_desc per settings toggles
     - For variable products: merge all variation attributes into one flat list
     - Strip HTML, trim to 2000 chars
   - `static store_embedding( int $product_id, array $vector, string $model ): void`
     - Single place to write both meta keys — called by `embed_product()` and batch AJAX handler
     - `update_post_meta( $product_id, '_wup_embedding', wp_json_encode( $vector ) )`
     - `update_post_meta( $product_id, '_wup_embedding_model', $model )`
   - `static embed_product( int $product_id ): bool`
     - Call `build_embed_text()`, then `WUP_Embedding_Client::make()->embed( $text )`
     - Call `store_embedding( $product_id, $vector, $current_model )` — do NOT duplicate meta writes
     - Return true on success, false on API error
   - `static get_embedding( int $product_id ): ?array`
     - Read + JSON-decode `_wup_embedding` meta
     - Return null if not yet embedded or model mismatch (`_wup_embedding_model` !== active model)

2. **Hook on product save** — enqueue async job, never block admin save:
   ```php
   // Use woocommerce_after_product_object_save (not woocommerce_update_product) —
   // ensures tags/attributes/meta are fully committed before building embed text
   add_action( 'woocommerce_after_product_object_save', function( WC_Product $product ) {
       if ( ! get_option( 'wup_ai_api_key' ) ) return;
       $product_id = $product->get_id();
       // Deduplicate: don't queue if already pending
       if ( as_has_scheduled_action( 'wup_embed_product', [ 'product_id' => $product_id ], 'wup-ai' ) ) return;
       as_schedule_single_action( time(), 'wup_embed_product', [ 'product_id' => $product_id ], 'wup-ai' );
   } );
   add_action( 'wup_embed_product', [ WUP_Product_Embedder::class, 'embed_product' ] );
   ```
   **Why `woocommerce_after_product_object_save` over `woocommerce_update_product`:**
   - `woocommerce_update_product` can fire before meta/terms are fully written in some WC versions
   - `woocommerce_after_product_object_save` guarantees all product data is committed

   **Why `as_schedule_single_action` + `as_has_scheduled_action` over `as_enqueue_async_action`:**
   - `as_enqueue_async_action` creates a new action every call — rapid saves = duplicate embed jobs
   - This pattern deduplicates: skip if already queued

## Success Criteria

- [ ] Saving a product triggers embedding generation
- [ ] `_wup_embedding` meta exists after save with valid 1536-element JSON array
- [ ] `_wup_embedding_model` meta stores the model name used
- [ ] `WUP_Product_Embedder::get_embedding( $id )` returns float[] for embedded products
- [ ] `get_embedding()` returns null when model has changed (forces re-embed)
- [ ] Variable product: embedding text includes all variation attributes merged
- [ ] No blocking delay on product save (async)
- [ ] Graceful no-op when API key not configured
